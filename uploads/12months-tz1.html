01
Продуктова концепція

Три напрямки —
одна екосистема

🎨
Концепція 01
Конструктор Почуттів

Користувач не вибирає з каталогу, а "збирає" букет у 2D-інтерфейсі. Ціна оновлюється в реальному часі. AI-флорист підказує гармонійні доповнення.

  • Вибір основи: крафт / коробка / кошик
  • Додавання квітів поштучно з live-ціною
  • AI аналізує склад і дає одну пораду
  • Підвищує середній чек на 30–40%
📅
Концепція 02
Квітковий Календар

Особистий кабінет із важливими датами. Бот нагадує за 3 дні з персональною добіркою на основі попередніх замовлень. Підписки.

  • Збереження дат: день народження, річниця
  • Push за 3 та 1 день до події
  • Добірка з урахуванням історії покупок
  • Абонемент: букет щотижня / щодва тижні
💘
Концепція 03
Квітковий Тіндер

Гейміфікований вибір через свайпи. Після 10–15 свайпів AI визначає смак і пропонує ідеальний букет. Відео-листівки через QR-код.

  • Свайп вправо / вліво без реєстрації
  • Алгоритм визначає смак: колір, форма, стиль
  • Відео або текстова листівка → QR у букеті
  • Точка входу: без тертя, без форм
🌿
Не обираємо одну концепцію — будуємо єдину екосистему. Тіндер — точка входу (легко, весело, без реєстрації). Конструктор — для просунутих, підвищує чек. Календар — утримання клієнта та повторні покупки протягом усіх 12 місяців року.

📡 Webhook vs Polling

Бот працює на webhook (не polling). При старті FastAPI-сервер реєструє webhook у Telegram. Це критично для продакшну — polling не підходить при великому навантаженні та не працює зі сторонніми хостами.

🔄 TWA ↔ Bot зв'язок

TWA передає дані боту через Telegram.WebApp.sendData(). Для підтвердження оплати та статусу — бот надсилає повідомлення напряму в чат користувача через bot.send_message().

Основні UX-флоу
🚀
Онбординг нового користувача
01
Бот вітається
Коротке привітання. Три кнопки: "Підібрати букет", "Зібрати свій", "Нагадування про дати". Без реєстрації.
02
Одне питання
Для кого букет? 5 варіантів: Коханій / Мамі / День народження / В офіс / Просто так. Зберігається у профіль.
03
TWA відкривається миттєво
Персоналізований контент відразу. Тіндер-вибір з відфільтрованим каталогом під привід.
04
Замовлення + підтвердження
Вибір → Кошик → Checkout → Оплата → Бот надсилає підтвердження в чат з трекінгом.
💌
Відео-листівка + QR-код
01
Пропозиція при оформленні
На кроці checkout: "Додати особисте привітання?" Не обов'язково — є кнопка "Пропустити".
02
Запис або текст
TWA Camera API для відео (до 30 сек) або текстова листівка з вибором фону.
03
Генерація QR на бекенді
Backend: унікальний UUID → URL → Python qrcode → PNG. Флорист друкує, вкладає в букет.
04
Отримувач сканує
Відкривається TWA з привітанням у стилі 12 Months. Без зайвих кроків. Можна переглянути ще раз.
Флоу оформлення замовлення
1
Підтвердження
Склад, кількість, вартість. Кнопка "Редагувати" повертає до каталогу.
2
Доставка
Самовивіз або доставка. Геолокація TWA для адреси. Слот часу через inline-календар.
3
Отримувач
Ім'я та телефон. Відео-листівка (опційно). Коментар до замовлення.
4
Оплата
LiqPay або Monobank. Підтвердження — бот пише в чат. Статус оновлюється live.

⚠️ Захист від помилок при видаленні

Перед видаленням позиції бот перевіряє наявність активних замовлень. Якщо є — пропонує лише "Зняти з продажу" замість повного видалення. Після підтвердження видалення — дані зберігаються в архіві 30 днів для звітності.

📦 Ранковий режим наявності (/stock)

Один екран зі всіма позиціями. Флорист тапає ✅/❌ навпроти кожної. Одне натискання "Зберегти" — всі зміни застосовуються одразу. Замість переходів по підменю — максимально швидко.

💡 Важливо: Cost Control

Claude API викликається тільки при реальній потребі (не на кожен крок). Конструктор — debounce 1.5s. Тіндер — один виклик після сесії. Нагадування — один виклик на подію. Середній витрата: ~$0.001 на сесію. Для MVP — Claude Haiku (найдешевший), при масштабуванні — Sonnet.
